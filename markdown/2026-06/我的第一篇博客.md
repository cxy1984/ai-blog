# 我的第一篇博客

今天开始搭建自己的技术博客。

这个博客使用 Markdown 写文章，然后通过项目里的 Node.js 脚本生成静态 HTML 页面，最后部署到 GitHub Pages。以后我只需要专注写 Markdown，构建脚本会自动更新首页、文章详情页、文章概览和站点地图。

## 为什么要写博客

写博客的目标不是追求一次写得多完整，而是把日常学习、排查问题、阅读源码、配置环境的过程沉淀下来。

我准备先记录这些内容：

- Java 和 Spring 学习笔记
- 开发环境配置过程
- 常见报错和解决方案
- 项目实践中的复盘
- AI 工具辅助开发的经验

## 这个博客怎么更新

每次新增文章时，只需要在 `markdown/YYYY-MM/` 目录下创建一个 `.md` 文件。例如：

```text
markdown/2026-06/我的第一篇博客.md
```

写完文章后执行：

```powershell
npm run build
```

构建完成后提交并推送：

```powershell
git add .
git commit -m "add first blog"
git push
```

GitHub Pages 会自动使用仓库里的静态文件发布网站。
